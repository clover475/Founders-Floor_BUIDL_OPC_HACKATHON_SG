# Hackathon Compliance

Source: BUIDL_OPC_Hackathon_SG 2026 Participant Handbook, reviewed 2026-07-12.

## Hard constraints

- Hacking window: 11:00–18:00 SGT on 2026-07-12.
- All submitted code must be written during this period.
- Pre-event ideas, research, wireframes, designs, and user flows are allowed.
- Existing projects, prior codebases, and prebuilt custom components are not allowed as a basis.
- Open-source libraries and frameworks are allowed with license checks and credits.
- Generative AI coding tools are allowed.
- Version control must be used throughout; commit history matters.
- Submission after 18:00 is not accepted.

## Required submission content

- Source code.
- Project name and description.
- Explanation of relevance to OPC / Agentic Services.
- Team information.
- Credits for contributors, libraries, repositories, and third-party assets.

## Repository practice

- Commit the bootstrap and docs first.
- Commit each working vertical milestone separately.
- Never squash the hackathon history before judging.
- Avoid a single large final commit.
- Tag the submitted state after the final pre-deadline commit.

## Suggested commit sequence

1. `docs: define hackathon scope and architecture`
2. `chore: scaffold Next.js application`
3. `feat: add demo data and local storage boundary`
4. `feat: complete clock-in and office presence flow`
5. `feat: add desk-check interaction`
6. `feat: ship clock-out and ship wall loop`
7. `feat: add break room and elevator pitch`
8. `docs: finalize demo, credits, and submission notes`

## Submission checklist

- [ ] Public or judge-accessible repository URL works.
- [ ] Commit history visibly spans the hacking period.
- [ ] README explains setup and demo flow.
- [ ] All external assets and libraries are credited.
- [ ] Product description explicitly connects to OPC and agentic services.
- [ ] Deployed demo works in a clean browser.
- [ ] Backup screen recording exists.
- [ ] OpenArena entry is submitted before 18:00 SGT.
